## What this example shows

The foundational Loop Engine pattern. An expense submission enters a two-step approval loop. A human-only guard on the approval transition ensures automation and AI actors can never self-approve.

## Loop diagram

```text
SUBMITTED --------[submit_expense]-------> PENDING_APPROVAL
PENDING_APPROVAL ------[approve]---------> APPROVED (terminal)
PENDING_APPROVAL -------[reject]---------> REJECTED (terminal)
```

## Actors

| Actor | Type | Transitions | Guards |
|---|---|---|---|
| submitter | human or automation | `submit_expense` | none |
| approver | human | `approve`, `reject` | `human-only` (hard) |

## Key annotated snippet

```ts
import { LoopBuilder } from "@loop-engine/sdk";

// The human-only guard is the load-bearing governance rule.
const expenseApproval = LoopBuilder
  .create("expense.approval", "finance")
  .description("Expense report approval")
  .state("SUBMITTED")
  .state("PENDING_APPROVAL")
  .state("APPROVED", { isTerminal: true })
  .state("REJECTED", { isTerminal: true })
  .initialState("SUBMITTED")
  .transition({
    id: "submit_expense",
    from: "SUBMITTED",
    to: "PENDING_APPROVAL",
    actors: ["human", "automation"]
  })
  .transition({
    id: "approve",
    from: "PENDING_APPROVAL",
    to: "APPROVED",
    actors: ["human"],
    guards: [
      {
        id: "human-only" as never,
        description: "Only a human approver can approve expenses",
        severity: "hard",
        evaluatedBy: "runtime"
      }
    ]
  })
  .transition({ id: "reject", from: "PENDING_APPROVAL", to: "REJECTED", actors: ["human"] })
  .build();
```
