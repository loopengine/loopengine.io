## What this example shows

Fraud queues move faster than human teams can triage manually. This pattern uses AI scoring for prioritization and keeps final decision authority with a human analyst. Every escalation and disposition remains attributable for audit.

## Loop diagram

```text
IDLE --[ingest_flag]--> FLAGGED --[start_scoring]--> SCORING
SCORING --[submit_score]--> PENDING_REVIEW
PENDING_REVIEW --[dismiss_case]--> DISMISSED (terminal)
PENDING_REVIEW --[analyst_review]--> REVIEWED --[close_case]--> CLOSED (terminal)
```

## Actors

| Actor | Type | Role | Guards that apply |
|---|---|---|---|
| transaction ingestor | automation | Flags incoming transactions | none |
| risk scorer | ai-agent | Computes fraud likelihood and patterns | `evidence-required`, `confidence-threshold` |
| analyst reviewer | human | Final disposition on escalated cases | `human-only` |

## Key annotated snippet

```ts
// @no-typecheck
import OpenAI from "openai";
import { createOpenAIActorAdapter } from "@loop-engine/adapter-openai";

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
const scorer = createOpenAIActorAdapter(openai, { modelId: "gpt-4o" });

const { actor, decision } = await scorer.createSubmission({
  loopId: "fraud.review",
  loopName: "Fraud Review",
  currentState: "SCORING",
  availableSignals: [{ signalId: "submit_score", name: "Submit Score" }],
  instruction: "Score fraud risk and provide suspicious patterns.",
  evidence: { merchantId: "m-42", ipRisk: "high", velocitySpike: true }
});

await engine.transition({
  aggregateId: "txn-884422" as never,
  transitionId: "submit_score" as never,
  actor,
  evidence: {
    ...decision,
    fraud_score: decision.confidence,
    flagged_patterns: ["velocity_spike", "new_device", "geo_mismatch"],
    transaction_context: { amount: 1480, currency: "USD" }
  }
});
```
