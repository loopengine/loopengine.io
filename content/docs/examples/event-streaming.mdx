## What this example shows

Every transition emits a typed event. This example shows how to subscribe, persist an audit stream, and reconstruct a timeline from transition history for replay and debugging.

## Loop diagram

```text
START ----> TRANSITION_1 ----> TRANSITION_2 ----> TERMINAL
  |              |                  |
  +---- emit events on every transition ----+
```

## Actors

| Actor | Type | Transitions | Guards |
|---|---|---|---|
| runtime emitter | automation | all runtime transitions | loop-defined guards |
| subscribers | automation | none (read-only stream consumers) | n/a |

## Key annotated snippet

```ts
import { LOOP_EVENT_TYPES } from "@loop-engine/events";
import { buildTimeline, replayLoop } from "@loop-engine/observability";
import { aggregateId } from "@loop-engine/core";

const audit: Array<{ at: string; type: string; aggregateId: string }> = [];

eventBus.subscribe(async (event) => {
  if (
    event.type === LOOP_EVENT_TYPES.LOOP_STARTED ||
    event.type === LOOP_EVENT_TYPES.TRANSITION_EXECUTED ||
    event.type === LOOP_EVENT_TYPES.LOOP_COMPLETED
  ) {
    audit.push({
      at: event.occurredAt,
      type: event.type,
      aggregateId: String(event.aggregateId)
    });
  }
});

const id = aggregateId("EXP-1");
const instance = await engine.getState(id);
const history = await engine.getHistory(id);
if (instance) {
  const timeline = buildTimeline(instance, history);
  const replay = replayLoop(definition, history);
  console.log(timeline.durationMs, replay.valid);
}
```
