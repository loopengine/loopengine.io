## What this example shows

Not every loop needs AI. This example models rule-based detection where an automation actor watches metrics and emits a start signal when thresholds are crossed. It is the SENSE stage in the operational loop pattern.

## Loop diagram

```text
METRIC_STREAM --[threshold_breach]--> SIGNAL_DETECTED
SIGNAL_DETECTED --[start_loop]--> LOOP_STARTED
LOOP_STARTED --[run_governed_flow]--> COMPLETE
```

## Actors

| Actor | Type | Transitions | Guards |
|---|---|---|---|
| detector | automation | `threshold_breach`, `start_loop` | none |
| runtime | system | `loop_start` | none |

## Key annotated snippet

```ts
import { createSignalEngine, thresholdBreachRule } from "@loop-engine/signals";
import { aggregateId } from "@loop-engine/core";

const signalEngine = createSignalEngine();
signalEngine.registerRule(
  thresholdBreachRule({
    field: "demandChange",
    operator: "gt",
    threshold: 0.75
  })
);

signalEngine.subscribe(async (signal) => {
  await engine.start({
    loopId: "scm.replenishment",
    aggregateId: aggregateId(`repl-${Date.now()}`),
    actor: { type: "automation", id: "system:signal-detector" },
    metadata: {
      sourceSignalType: signal.type,
      sourceSignalId: signal.id,
      sku: signal.payload?.sku
    }
  });
});
```
