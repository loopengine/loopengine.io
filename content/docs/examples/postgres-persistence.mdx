## What this example shows

Loop Engine storage is an adapter boundary. This example swaps `adapter-memory` for `adapter-postgres` without changing loop definitions or transition logic.

## Loop diagram

```text
START LOOP ----> WRITE INSTANCE (Postgres)
TRANSITION ----> APPEND HISTORY (Postgres)
QUERY STATE ---> READ INSTANCE + READ HISTORY
```

## Actors

| Actor | Type | Transitions | Guards |
|---|---|---|---|
| service runtime | automation | all business transitions | configured in loop definition |
| reviewer (optional) | human | approval transitions | `human-only`, policy guards |

## Key annotated snippet

```ts
import { createLoopSystem } from "@loop-engine/sdk";
import { postgresStore, createSchema } from "@loop-engine/adapter-postgres";
import { memoryStore } from "@loop-engine/adapter-memory";
import { Pool } from "pg";

// Version A: in-memory (default local dev)
const memStore = memoryStore();
const memoryRuntime = await createLoopSystem({ loops: [definition], store: memStore });

// Version B: postgres (durable)
const pool = new Pool({ connectionString: process.env.DATABASE_URL });
await createSchema(pool);
const pgStore = postgresStore(pool);
const postgresRuntime = await createLoopSystem({ loops: [definition], store: pgStore });

// Loop definitions + transition calls are unchanged across adapters.
```
