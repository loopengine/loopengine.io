`@loop-engine/adapter-kafka` implements the runtime `EventBus` interface on top of a Kafka-like producer/consumer client.

## Install

```bash
npm install @loop-engine/adapter-kafka kafkajs
```

## Event bus factory

```ts
kafkaEventBus(options: {
  kafka: KafkaLike
  topic: string
  groupId?: string
}): EventBus
```

- `emit()` sends serialized `LoopEvent` payloads to `options.topic`.
- `subscribe()` starts a consumer with `groupId` (default: `loopengine`) and forwards parsed messages to the handler.

```ts
import { Kafka } from "kafkajs"
import { kafkaEventBus } from "@loop-engine/adapter-kafka"

const kafka = new Kafka({ brokers: ["localhost:9092"] })
// kafkajs `producer().send` return type is wider than the adapter's `KafkaLike` stub; emit path is compatible.
const bus = kafkaEventBus({ kafka: kafka as any, topic: "loop-events", groupId: "loopengine-docs" })
```

## Topic strategy

Use one topic per environment or domain boundary to keep replay and retention policies explicit.
