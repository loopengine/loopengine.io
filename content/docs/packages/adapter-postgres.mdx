`@loop-engine/adapter-postgres` provides schema setup and a PostgreSQL store interface target for runtime persistence.

## Install

```bash
npm install @loop-engine/adapter-postgres pg
```

## Schema bootstrap

```ts
createSchema(pool: PgPoolLike): Promise<void>
```

`createSchema()` creates:

- `loop_instances`
- `loop_transitions`

```ts
import { Pool } from "pg"
import { createSchema } from "@loop-engine/adapter-postgres"

const pool = new Pool({ connectionString: process.env.DATABASE_URL })
await createSchema(pool)
```

## Store factory

```ts
postgresStore(pool: PgPoolLike): LoopStore
```

<Callout variant="warning" title="Implementation status">
All `postgresStore` runtime methods are currently stubbed and throw `postgresStore runtime implementation pending`:
`getInstance`, `saveInstance`, `getTransitionHistory`, `saveTransitionRecord`, and `listOpenInstances`.
</Callout>

## Production notes

- Use `pg` connection pooling.
- Keep SSL, idle timeout, and max connection settings explicit in deployment config.
- Treat this package as schema-ready but runtime-incomplete in `v0.1.0`.
