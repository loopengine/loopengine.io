## What it does

PagerDuty incidents can trigger Loop Engine workflows, turning urgent alerts into governed and attributable response loops. The adapter maps incident events to loop signals, starts a loop per incident, and keeps human decisions in-policy.

## Use case

PagerDuty fires a `SEV-2` database incident:

1. Loop Engine starts `incident-response` loop
2. On-call engineer acknowledges (human-only guard)
3. Runbook automation executes
4. Loop closes and emits learning signal

## Quick setup

1. **Install**

   ```bash
   npm install @loop-engine/adapter-pagerduty
   ```

2. **Configure webhook endpoint**

   ```text
   POST https://your-app.example.com/hooks/pagerduty
   ```

3. **Wire adapter**

   ```typescript
   import { PagerDutyAdapter } from "@loop-engine/adapter-pagerduty";

   const pagerduty = new PagerDutyAdapter({
     webhookSecret: process.env.PAGERDUTY_WEBHOOK_SECRET!,
     incidentLoopId: "incident-response"
   });
   ```

4. **Define incident loop**

   ```yaml
   loopId: incident-response
   initialState: triage
   states:
     - stateId: triage
       transitions:
         - signalId: acknowledge_incident
           toState: executing
   ```

## Configuration reference

| Option | Type | Default | Description |
|---|---|---|---|
| `webhookSecret` | `string` | required | PagerDuty webhook signing secret |
| `incidentLoopId` | `string` | required | Loop definition ID to start on incident |
| `severityMap` | `object` | — | Map PagerDuty severity to loop signals |

## Troubleshooting

- **Webhook signature invalid** — Ensure `webhookSecret` matches PagerDuty integration settings.
- **Loop not starting** — Verify `incidentLoopId` is registered in runtime.
