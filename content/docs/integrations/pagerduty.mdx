## Overview

PagerDuty is the natural approval channel for enterprise AI actions because the on-call engineer already owns high-impact operational decisions. Loop Engine's `PENDING_HUMAN_APPROVAL` state maps directly to an incident: the system must pause until a human decides.

`@loop-engine/adapter-pagerduty` pages the right responder using existing escalation policy, severity routing, and acknowledgement workflows. Teams stay inside PagerDuty while Loop Engine preserves state-machine constraints and audit history.

## Demo scenario (purchase order)

Use this scenario for partner outreach: AI requests a `$9,200` PO, PagerDuty incident triggers, on-call engineer approves, loop executes, incident resolves.

- Incident title: `AI requested $9,200 PO to Acme Medical Devices`
- Approval link deep-links to loop decision UI
- Decision closes both loop state and PagerDuty incident

## How it works

1. AI tool call starts and `requiresApproval()` evaluates input risk.
2. Loop transitions to `PENDING_HUMAN_APPROVAL`.
3. `PagerDutyNotifier.notify()` triggers an incident.
4. On-call engineer opens incident and follows approval URL.
5. Human decision advances loop to `EXECUTING` then `EXECUTED` (or `REJECTED`).
6. `PagerDutyNotifier.resolve()` auto-resolves the incident.

## Setup

1. Create a PagerDuty service with an Events API v2 integration.
2. Copy the integration key.
3. Set `PAGERDUTY_INTEGRATION_KEY` in your runtime environment.
4. Follow PagerDuty integration docs for service setup:
   - [PagerDuty Events API v2](https://developer.pagerduty.com/api-reference/b3A6Mjc0ODE0MQ-send-an-event-to-pager-duty)

## Quick start

```ts
import { PagerDutyNotifier } from "@loop-engine/adapter-pagerduty";
import { wrapTool } from "@loop-engine/adapter-vercel-ai";

const pagerduty = new PagerDutyNotifier({
  integrationKey: process.env.PAGERDUTY_INTEGRATION_KEY!,
  severity: "warning", // can also be a function by input context
  serviceContext: "Loop Engine / Procurement Agent",
  approvalUrl: (loopId) => `https://app.betterdata.co/loops/${loopId}`
});

const submitPurchaseOrder = wrapTool(purchaseOrderTool, {
  loopDefinition: purchaseOrderLoop,
  engine,
  requiresApproval: ({ amount }) => amount > 5000,
  onApprovalRequired: async (loopId, input) => {
    const dedupKey = await pagerduty.notify(loopId, {
      loopId,
      loopName: "purchase-order",
      actor: { type: "ai-agent", id: "procurement-agent" as never },
      input,
      summary: `AI requested $${input.amount} PO to ${input.vendor}`
    });
    storeDedupKey(loopId, dedupKey);
  },
  actor: { type: "ai-agent", id: "procurement-agent" as never }
});

// Later, after human decision:
await pagerduty.resolve(dedupKey, "approved");
```

## Severity routing

Use severity routing to map action impact to escalation policies:

```ts
const pagerduty = new PagerDutyNotifier({
  integrationKey: process.env.PAGERDUTY_INTEGRATION_KEY!,
  severity: (context) => {
    const amount = Number((context.input as { amount?: number }).amount ?? 0);
    return amount > 50000 ? "critical" : amount > 5000 ? "warning" : "info";
  }
});
```

Example policy tiers:

- `critical` for actions over `$50k`
- `warning` for `$5k–$50k`
- `info` for under `$5k`

## API reference

| API | Description |
|---|---|
| `new PagerDutyNotifier(options)` | Creates notifier bound to one PagerDuty integration key. |
| `options.integrationKey` | PagerDuty Events API v2 routing key. |
| `options.severity` | Static severity or `(context) => severity` function. |
| `options.serviceContext` | Incident source label (default `Loop Engine`). |
| `options.approvalUrl(loopId)` | Optional deep-link to approval UI. |
| `options.customDetails(context)` | Additional custom details merged into incident payload. |
| `notify(loopId, context)` | Triggers incident and returns `dedupKey`. |
| `resolve(dedupKey, resolution)` | Resolves incident when loop leaves pending state. |
