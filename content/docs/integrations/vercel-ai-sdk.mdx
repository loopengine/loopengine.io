## What it does

Vercel AI SDK handles streaming and model access. Loop Engine governs when and whether model-requested tool actions are authorized. Use Vercel AI SDK for model execution; use Loop Engine for guard enforcement, actor attribution, and audit trails.

## Quick setup

1. **Install**

   ```bash
   npm install @loop-engine/adapter-vercel-ai ai
   ```

2. **Wrap a tool with governance**

   ```typescript
   import { z } from "zod";
   import { streamText, tool } from "ai";
   import { actorId, type ActorRef } from "@loop-engine/core";
   import { wrapTool, type CoreTool } from "@loop-engine/adapter-vercel-ai";

   // `wrapTool` expects the adapter's `CoreTool` shape; bridge the Vercel AI `tool()` helper
   // by wrapping a thin CoreTool that delegates to the same work unit.
   const baseTool = tool({
     description: "Submit purchase order",
     inputSchema: z.object({ vendor: z.string(), amount: z.number() }),
     execute: async ({ vendor, amount }) => ({ orderId: "PO-0042", vendor, amount })
   });
   const submitPurchaseOrder = wrapTool(baseTool as unknown as CoreTool<{ vendor: string; amount: number }, { orderId: string; vendor: string; amount: number }>, {
     loopDefinition: purchaseOrderLoop,
     engine,
     actor: {
       type: "ai-agent",
       id: actorId("ai:vercel-tool"),
       modelId: "gpt-4o",
       provider: "openai"
     } as ActorRef,
     requiresApproval: (input) => input.amount > 5000
   });
   ```

3. **Run with streaming**

   ```typescript
   await streamText({
     model: openai("gpt-4o"),
     tools: { submitPurchaseOrder },
     prompt: "Submit a PO to Acme Medical for $9,200"
   });
   ```

## How it works

1. Model requests tool invocation
2. Adapter starts and advances loop transitions
3. Approval gates hold high-impact actions
4. Human decision unblocks execution
5. Execution and evidence are appended to history

## Configuration reference

| Option | Type | Default | Description |
|---|---|---|---|
| `loopDefinition` | `LoopDefinition` | required | Governance state machine for the tool |
| `engine` | `LoopEngine` | required | Runtime instance |
| `requiresApproval` | `(input) => boolean` | — | Predicate for human checkpointing |
| `onApprovalRequired` | `(loopId, input) => Promise<void>` | — | Notification hook |
| `actor` | `Actor` | `ai-agent` | Actor attribution for transition records |

## Troubleshooting

- **Streaming parse errors** — Enforce strict JSON response contracts in tool output.
- **Model resolution errors** — Verify the selected model provider is available in your AI SDK config.
- **Missing approvals** — Ensure `requiresApproval` returns true for high-risk tool inputs.
